üöÄ Quick Start

Get up and running in minutes

1

Clone the repository

Terminal 
git clone https://github.com/Sakib-Dalal/cloudlab.git
cd cloudlab
2

Build the CLI

Terminal 
chmod +x build.sh
./build.sh

Or build manually:

go mod init cloudlab
go build -ldflags="-s -w" -o cloudlab cloudlab.go
sudo mv cloudlab /usr/local/bin/
3

Initialize (interactive setup)

Terminal
cloudlab init
4

Install all components (including SSH terminal)

Terminal
cloudlab install all
5

Start everything

Terminal
cloudlab start all

URLs will be sent to your email if configured!

6

Check status

Terminal
cloudlab status

üîí SSH Terminal Setup

Browser-based terminal access using ttyd

Installation

ttyd is automatically installed when you run:

cloudlab install all
# or
cloudlab install ssh

Manual Installation

macOS

brew install ttyd

Ubuntu/Debian

sudo apt-get install ttyd

Other Linux (binary)

wget https://github.com/tsl0922/ttyd/releases/latest/download/ttyd.x86_64
chmod +x ttyd.x86_64
sudo mv ttyd.x86_64 /usr/local/bin/ttyd

Usage 

# Start SSH terminal
cloudlab start ssh

# Access locally
# Open http://localhost:2222 in your browser

# Get public URL via Cloudflare tunnel
cloudlab tunnel start

# Check status
cloudlab ssh status

Deploy CloudLab on cloud instances

AWS EC2 / Google Cloud / Azure

SSH Terminal 
# SSH into your instance
ssh user@your-instance-ip

# Install Go
wget https://go.dev/dl/go1.21.5.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.5.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

cd cloudlab

# Initialize and start
cloudlab init
cloudlab install all
cloudlab start all

# Your SSH terminal is now accessible via Cloudflare tunnel!

Run on Startup (systemd)

Terminal 
sudo tee /etc/systemd/system/cloudlab.service << EOF
[Unit]
Description=CloudLab Services
After=network.target

[Service]
Type=forking
User=$USER
ExecStart=/usr/local/bin/cloudlab start all
ExecStop=/usr/local/bin/cloudlab stop all
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable cloudlab
sudo systemctl start cloudlab

üåê How Tunnels Work

CloudLab uses Cloudflare Quick Tunnels (TryCloudflare)

Benefits

  • No Cloudflare account required
  • Creates temporary public URLs
  • Free to use
  • Secure HTTPS connections
  • Separate tunnels for Jupyter, VS Code, and SSH

Note

  • URLs change when you restart tunnels
  • URLs are random like: random-words.trycloudflare.com

# To get new URLs:

cloudlab tunnel restart
cloudlab email send  # Send new URLs to your email
